Именование переменных

  • Имена
    • Имена должны содержать развернутое понятное описание переменной, без русских названий, без жаргонов, без лишних сокращений, если имя содержит несколько слов отделять их нижним_подчеркиваем.
    window_handler
    okonii_obrabotchik
    winHadler
    wndhdlr


  • Именнование констант
    • Имена констант писать ЗАГЛАВНЫМИ БУКВАМИ.
    TITLE_HEIGHT
    title_const


  • Именнование глобальных переменных
    • Имена глобальных переменных начинаються с Заглавной буквы, остальные слова в имени в зависимости от граматики.
    Global_variable
    global_var
    global_VAR


  • Именнование локальных переменных
    • Имена локальных переменных начинаються с прописной(маленькой) буквы, остальные слова в имени в зависимости от граматики.
    window_handler
    winHadler


  • Именнование параметров функции
    • Имена передаваемых параметров функций начианються с нижнего подчеркивания.
    _window_handler
    window_handler


  • Именнование полей класса
    • Имена полей класса начинаються с m_ ( означает member ), если поля статические то также как глобальные переменные.
    m_window_handler
    window_handler


  • Именнование функций и методов
    • Имена функций и методов с Заглавной буквы и между словами не имеют отступов, каждое следующее слово с Заглавной буквы.
    MyFunctionDestroyer
    my_functionDestroyer


  • Именнование классов и пространств
    • Каждый отдельный модуль имеет свое пространство имен которое создается по правилу, пример File Manager = f-file mng-manager -> fmng. Имена классов Также как функции и методы.

    Скобочки

  • Фигурные
    • Фигурные скобки стаяться на тойже сроке после кона выражения, исключение пустые скобки. 
    if( condition ) {
			
} else {

}

  • Конструктор иниициализации
    • Начинаются со следующей строчки. 
    MyClass::MyClass(int _a, int _b, int _c)
: m_a(_a), m_b(b), m_c(c) 
{}
